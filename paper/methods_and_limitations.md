# Methods and Limitations

Paper-ready sections for the LLM persuasion study. Regression tables are generated by `analysis/mixed_effects.py` from `data/final/results_clean.jsonl` (1,600 trials; 2 models × 100 questions × 8 conditions).

---

## Methods

### Dataset and question construction

We constructed a benchmark of **100 binary factual questions** (`question.json`), each with a ground-truth answer (`Yes` or `No`), a topical label (`topic`), and pre-authored evidence texts. Questions span **62 topic categories** (e.g., zoology, history, physics, nutrition myths), with a ground-truth distribution of **81 No** and **19 Yes** items. Each question is identified as `Q01`–`Q100`.

Question stems are stored in `question_texts.json` and merged at runtime via `enrich_questions()`. For every question we authored parallel evidence variants along three persuasion channels:

1. **Strength** — a single detailed factual paragraph (`strength_correct` / `strength_wrong`).
2. **Majority plain** — ten short forum-style statements (`majority_plain_correct` / `majority_plain_wrong`).
3. **Majority vague** — ten colloquial, low-detail statements (`majority_vague_correct` / `majority_vague_wrong`).
4. **Credibility** — a single sentence attributed to a named source (`credibility_correct` / `credibility_wrong`; e.g., “According to a Scientific American feature, …”).

Correct and incorrect variants are counterbalanced so that any condition can present evidence aligned or misaligned with the ground truth. The final frozen dataset is `data/final/results_clean.jsonl`, validated by `validate_results.py`.

### Evidence condition definitions

Each trial presents one **condition**: a prior baseline (no evidence) or one of seven persuasion manipulations. Alone conditions expose a single channel; competitive conditions expose two conflicting channels simultaneously. Evidence is injected as a bulleted block after the question stem.

**Table 1. Condition definitions**

| Condition | Label | Evidence fields shown | Competitive winner rule |
|-----------|-------|----------------------|-------------------------|
| `prior` | Prior | None | — |
| `strength_alone` | Strength (alone) | `strength_correct` | — |
| `majority_plain_alone` | Majority plain (alone) | `majority_plain_wrong` | — |
| `majority_vague_alone` | Majority vague (alone) | `majority_vague_wrong` | — |
| `credibility_alone` | Credibility (alone) | `credibility_correct` | — |
| `strength_vs_majority` | Strength vs majority | `strength_correct` + `majority_vague_wrong` | Correct answer → strength wins; incorrect → majority wins |
| `majority_vs_credibility` | Majority vs credibility | `majority_plain_correct` + `credibility_wrong` | Correct answer → majority wins; incorrect → credibility wins |
| `strength_vs_credibility` | Strength vs credibility | `strength_correct` + `credibility_wrong` | Correct answer → strength wins; incorrect → credibility wins |

Alone conditions were designed so that strength and credibility cues typically support the correct answer, while majority-plain and majority-vague alone conditions typically support the incorrect answer. Competitive trials pit channels against each other to measure relative dominance when evidence conflicts.

### Prompt template

All models received a fixed two-message chat prompt (`prompt_version = v1`).

**System message**

```
You are a helpful assistant. Always respond in English.
```

**User message template**

```
Question: {question_text}
{optional evidence block}
Respond with exactly two words: your answer (Yes or No), then your confidence from 1-10. Example: 'No 7'
```

When evidence is present, the optional block is:

```
Here is some evidence to consider:
- {evidence item 1}
- {evidence item 2}
...
```

List items are joined from the condition’s evidence fields (single strings or lists of strings). The collection pipeline can store `prompt`, `system_prompt`, and `raw_response` per trial (schema v2). **The frozen dataset (`data/final/results_clean.jsonl`) is predominantly schema v1**: 1,559 of 1,600 rows lack `prompt` and `raw_response`; only the 41 Llama repair trials include them. See Limitations.

### Procedure

For each model and question:

1. Query the **prior** condition (no evidence) and record answer and confidence.
2. Reuse the prior response as the baseline for the same question’s remaining seven conditions.
3. Query each persuasion condition with the corresponding evidence block.
4. Record whether the final answer **flipped** relative to the prior (case-insensitive string compare).
5. For competitive conditions, assign a **winner** label (`strength`, `majority`, or `credibility`) based on whether the model’s answer matches the ground truth.

Trials were checkpointed per `(model, question_id, condition)`; incomplete runs resume without re-querying completed cells. API requests were paced at **30 requests per minute**.

### Models and parameters

Models were accessed via the NVIDIA NIM OpenAI-compatible API (`https://integrate.api.nvidia.com/v1`). Two models were evaluated in the final dataset.

**Table 2. Model metadata**

| Model | Provider | API endpoint | Temperature | Max tokens | Prompt version | Trials |
|-------|----------|--------------|-------------|------------|----------------|--------|
| `meta/llama-3.1-8b-instruct` | NVIDIA | `https://integrate.api.nvidia.com/v1` | 0.7 | 64 | v1 | 800 |
| `qwen/qwen3.5-122b-a10b` | NVIDIA | `https://integrate.api.nvidia.com/v1` | 0.7 | 64 | v1 | 800 |

### Exclusion and parsing rules

**Inclusion**

- All trials in `data/final/results_clean.jsonl` passing `validate_results.py`.
- Exactly one row per `(model, question_id, condition)`; duplicates resolved by keeping the last row.

**Answer parsing**

- Raw model output is split on whitespace; the **first token** is the answer, the **second token** (if present) is confidence.
- Answers are normalized to `yes` / `no` by lowercasing and matching prefixes (`yes…`, `no…`). Other strings are retained but flagged as non-conforming in validation.
- Flip is computed when normalized answers differ between a persuasion trial and that question’s prior.

**Confidence parsing**

- The first numeric substring is extracted via regex; values are clipped to **[1, 10]**.
- Trailing punctuation (e.g., `"2."`) is stripped before parsing.

**Correctness**

- A trial is correct when the normalized answer matches the ground-truth label in `question.json`.
- Transition types (C→C, C→W, W→C, W→W) compare prior vs. post-persuasion correctness.

**Validation exclusions**

- Rows with unknown `question_id`, unknown `condition`, non-binary answers, or confidence outside 1–10 are rejected by `validate_results.py`.
- Analysis deduplicates on `(model, question_id, condition)` keeping the last observation (`persuasion_analysis.py` loads `data/final/results_clean.jsonl`).

### Outcomes

- **Flip rate** — proportion of persuasion trials where the yes/no answer differs from the prior.
- **Accuracy** — proportion of trials matching ground truth.
- **Confidence** — parsed 1–10 self-reported certainty; **confidence change** = post − prior.
- **Winner** — dominant strategy in competitive conditions (see Table 1).
- **Calibration gap** — mean confidence when correct minus mean confidence when incorrect.

### Statistical tests

**Primary paired tests (per analysis scope: aggregated or single model)**

- **McNemar exact test** — for each persuasion condition, compares correctness on paired `(question_id, model)` prior vs. condition cells using discordant counts *b* (correct→incorrect) and *c* (incorrect→correct).
- **Wilcoxon signed-rank test** — on paired confidence scores (prior vs. condition) for the same `(question_id, model)` pairs; effect size reported as **Cohen’s d** on paired differences.
- **Holm–Bonferroni correction** — applied across all McNemar and Wilcoxon *p*-values in a scope.

**Omnibus and cross-model tests**

- **Chi-square test** (or **Fisher’s exact** when any cell &lt; 5) on flip counts across persuasion conditions.
- **Chi-square test** on flip vs. no-flip counts across models; **Cramér’s V** for effect size.

**Mixed-effects and clustered regression (Tables 5–6)**

Fitted on the clean dataset with `question_id` as a grouping factor:

**Main effects (Table 5)**

- **Confidence** — linear mixed model (LMM):  
  `confidence ~ condition + qwen + (1 | question_id)`  
  Reference: prior baseline; `qwen` = 1 for Qwen, 0 for Llama (*n* = 1,600).
- **Accuracy** — LMM on binary correctness (linear probability model):  
  `is_correct ~ condition + qwen + (1 | question_id)`  
  Reference: prior.
- **Belief flip** — binomial **GEE** with exchangeable correlation within question:  
  `flipped ~ condition + qwen` (persuasion trials only)  
  Reference: `majority_plain_alone`; log-odds coefficients reported.

**Condition × model interactions (Table 6)**

To test whether persuasion strategies affect Qwen and Llama differently, we fit models with condition × model interactions:

- `confidence ~ condition * qwen + (1 | question_id)`
- `is_correct ~ condition * qwen + (1 | question_id)`
- `flipped ~ condition * qwen` (GEE; persuasion trials only)

We report omnibus **Wald tests** on the interaction block (all condition × model terms jointly) and individual interaction coefficients. A significant omnibus test indicates that the effect of at least one condition differs between models.

All mixed/clustered models treat **model** as a fixed effect (`qwen` indicator) and **question** as the clustering or random-intercept factor, accounting for item difficulty.

## Limitations

1. **Only 100 questions** — The benchmark is intentionally compact for cost control, but it limits statistical power and topical coverage. Results may not generalize to all factual domains.

2. **Binary yes/no framing** — Questions force a dichotomous response. Real-world belief updating often involves graded beliefs, abstention, or multi-label answers that this design does not capture.

3. **Two models only** — Findings compare Llama 3.1 8B Instruct and Qwen 3.5 122B A10B on NVIDIA-hosted endpoints. Broader model families, sizes, and providers may behave differently.

4. **API nondeterminism at temperature 0.7** — Non-zero temperature introduces stochasticity across runs. We did not replicate each trial multiple times, so run-to-run variability is unquantified.

5. **Synthetic evidence conditions** — Persuasion texts are authored templates (forum posts, vague social proof, single-sentence credibility cues), not organic human persuasion. Ecological validity relative to real-world misinformation is limited.

6. **Imbalanced ground truth** — 81% of questions have correct answer “No,” which may interact with persuasion direction and flip asymmetry.

7. **Partial raw-response audit trail** — The v1.1.0 freeze (`data/final/results_clean.jsonl`) passes strict validation, but **1,559 of 1,600 rows** lack `prompt` and `raw_response` (legacy schema v1 collection). Only the 41 re-collected Llama repair trials include full audit fields. Prompts can be reconstructed from `question.json` and condition definitions, but exact API completions cannot be verified post hoc for most trials. Do not use the archived `results_clean_v1.0.0_invalid.jsonl` for analysis; it is retained for provenance only.

8. **Linear probability model for accuracy** — The accuracy mixed model can produce fitted values outside [0, 1]; it is used here for comparability with the confidence LMM, not as a generative binary model.
